## 🤖 로봇 규격

| 구분 | 상세 사양 |
|------|-----------|
| **MCU** | ATmega 8-Bit 계열 (종류 무관, 로봇당 1개) |
| **크기** | 가로·세로 130mm 이내 / 높이 70mm 이내 |
| **구동계** | DC 모터 **2개** |
| **경로 센서** | IR 센서 **2개** (라인트레이싱) |
| **장애물 센서** | 전방 IR 센서 **3개 이내** (후방 장착 불가) |
| **Lifter** | 1~4점 지지 방식 (서보+기어박스 권장) |
| **RFID** | 13.56MHz 규격 Reader/Writer (NFC 모듈) |
| **전원** | Li-Po 배터리 (전압·암페어 제한 없음, 상시전원 불가) |

> ⚠️ 리프터 상단 적재부 바닥 높이는 **75mm 고정** (Pallet 베이스 높이와 일치)

## 🎮 경기 방식

### 미션 흐름

```
[대기] RFID 카드 태깅으로 출발
   ↓
[이동] 물류창고 좌표로 라인트레이싱
   ↓
[인식] Pallet RFID UID 확인
   ↓
[상차] Lifter UP → Pallet 들어올리기
   ↓
[이동] 행선지 좌표로 이동
   ↓
[하차] Lifter DOWN → Pallet 내려놓기  (+10점)
   ↓
└─ 5회 반복 (팔렛 5개)
```

### 주요 규칙

- 출발은 반드시 **RFID 카드 태깅**으로만 가능 (버튼 불가)
- 두 로봇은 각각 **별도의 RFID 카드**로 출발
- 행선지 순서는 **임파운딩 이후 미션으로 공개** (숨겨진 미션 포함 가능)
- 경기 중 로봇 임의 터치 시 **Pickup Penalty 적용**

## 📊 채점표

### 득점

| 항목 | 배점 |
|------|------|
| 정상 물류 이동 (1개당) | **+10점** |
| 최대 득점 (5개 × 10점) | **50점** |

### 감점

| 항목 | 감점 |
|------|------|
| 장애물 충돌 (1개당) | **-2점** |
| Pickup Penalty | **-5점** |
| Full Reset Penalty | **감점 없음** (1회 한정) |

### 동점자 처리 순서

1. 두 주행 중 **높은 점수** 비교
2. 높은 점수도 동점 → **낮은 점수** 비교
3. 모두 동점 → 장애물 충돌 수 → Pickup Penalty → Full Reset Penalty 순
4. 모두 동일 → **빠른 완주 시간** (타임오버 제외)

## 💻 소스코드 설명

이 레포지토리에는 동일한 목적을 수행하는 **3가지 버전**의 아두이노 코드가 포함되어 있습니다.

### 버전 비교

| | v1 — 좌표계 | v2 — 상태머신 | v3 — 교차점 카운팅 |
|---|---|---|---|
| **이동 원리** | 행/열 좌표 자동 계산 | 시간 기반 격자 이동 | 교차점 물리 감지 |
| **IR 센서 수** | 2개 | 2개 | **3개** (중앙 추가) |
| **오차 누적** | 중간 | 높음 | **없음** |
| **경기 당일 수정** | 좌표 배열 | 문자열 명령 | 구조체 배열 |
| **개별 테스트** | 시리얼 로그 | ✅ 시리얼 명령어 | ✅ 시리얼 명령어 |
| **추천 상황** | 복잡한 맵 | 빠른 디버깅 | **가장 안정적** |

---

### v1 — 좌표계 자동 경로 (`v1_Coordinate.ino`)

격자 좌표 `(row, col)` 을 입력하면 맨해튼 경로로 자동 이동합니다.

```cpp
// 경기 당일 이것만 수정
Coord WAREHOUSE[5]    = {{1,1}, {1,3}, {2,1}, {3,2}, {0,2}};
Coord DESTINATION[5]  = {{0,4}, {4,0}, {0,4}, {2,4}, {4,0}};
byte  START_UID[4]    = {0x11, 0x22, 0x33, 0x44};
```

---

### v2 — 상태머신 (`v2_StateMachine.ino`)

이동 경로를 문자열 명령어로 직관적으로 작성합니다.

```
F = 앞으로 1칸    L = 좌회전    R = 우회전
B = 유턴(180°)   U = Lifter UP  D = Lifter DOWN
W = RFID 확인    P = 짧은 정지
```

```cpp
// 경기 당일 이것만 수정
const char* MISSION_CMDS[5] = {
  "FFWU" "RFF" "D",      // 팔렛 1
  "BFFW" "U" "RFFFD",    // 팔렛 2
  ...
};
```

시리얼 모니터 명령으로 각 동작을 개별 테스트할 수 있습니다.

```
T → 전진   L → 좌회전   R → 우회전
U → 리프터 UP   D → 리프터 DOWN   S → 정지
```

---

### v3 — 교차점 카운팅 (`v3_IntersectionCount.ino`)

3개 IR 센서로 교차점을 물리적으로 감지해 칸 수를 셉니다. 시간 오차가 없어 가장 정밀합니다.

```cpp
// 경기 당일 이것만 수정
const MoveCmd MISSION[] = {
  {2, 2, 'W'},  // 우회전 후 2칸 이동, RFID 확인
  {0, 0, 'U'},  // 제자리 Lifter UP
  {3, 3, 'N'},  // 유턴 후 3칸 직진
  {1, 2, 'D'},  // 좌회전 2칸 후 Lifter DOWN
  ...
};
```

## ⚙️ 현장 보정 파라미터

경기 당일 로봇을 조립한 뒤 아래 값을 실측하여 수정하세요.

```cpp
// ── v2 보정 ──────────────────────────────
#define TIME_ONE_GRID  600   // 1격자 이동 시간 (ms)
#define TIME_TURN_90   470   // 90도 회전 시간 (ms)

// ── v3 보정 ──────────────────────────────
#define LINE_TH        450   // IR 라인 감지 임계값 (300~700)
#define INTER_DELAY    150   // 교차점 중심 정렬 지연 (ms)
#define TURN_90_MS     500   // 90도 회전 시간 (ms)

// ── 공통 ─────────────────────────────────
#define LIFTER_UP_DEG  88    // 리프터 상승 각도 (서보)
#define LIFTER_DN_DEG  3     // 리프터 하강 각도 (서보)
#define SPEED_BASE     150   // 직진 기본 속도 (0~255)
```

> **보정 순서 권장:** 리프터 각도 → 회전 시간 → 직진 속도 → 라인 임계값

## ⚠️ 주요 반칙 규정

| 반칙 | 처리 |
|------|------|
| 심판 Start 후 로봇 터치 | **Pickup Penalty (-5점)** |
| 장애물 건드림 | **-2점 × 개수** |
| Full Reset 요청 | 감점 없음, 단 1회 한정 / 이전 점수 초기화 |
| 부분 리셋 | **허용 안 됨** |
| 코치의 경기장 입장 | 실격 가능 |
